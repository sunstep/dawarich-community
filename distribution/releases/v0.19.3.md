## Highlights

This release focuses on production stability and usability improvements. Experimental tracker intelligence and runtime mode switching are not included — the reliable GPS stream tracker remains unchanged.

### What's new

- **Timeline — local points connect to API track**: local (unsynced) points now stitch seamlessly into the existing API track instead of floating disconnected. After a batch upload the merged points integrate immediately without a network refresh.
- **Timeline — distance filter preference**: the minimum distance between recorded points can now be set per-user in Settings, letting you reduce noise on slow or stationary periods.
- **Timeline — historical date fix**: the current-location indicator no longer appears on historical date views where it was confusing and out of sync.
- **Timeline — correct day boundary**: the last-day timestamp calculation now uses the maximum timestamp, preventing cutoff artefacts at day boundaries.
- **Stats — offline fallback**: refreshing stats while offline no longer wipes the cached stats. The last fetched values are preserved until a successful remote fetch.
- **Auth — QR scan on hosted instances**: the QR scanner is now available on the hosted/cloud login screen, not only on self-hosted setups.
- **Onboarding — skip permissions**: the permissions onboarding screen can now be dismissed without granting every permission. A note explains which features require them.
- **Self-signed certificates**: HTTP traffic now routes through a native HTTP client that respects user-installed root CAs, enabling self-signed certificate support.
- **Batch upload expiration**: uploads older than a configurable threshold are now automatically expired and cleaned up.
- **Auto mode as default**: new installs default to GPS stream (auto) mode instead of fixed-interval timer mode.
